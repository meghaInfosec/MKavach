## A.5 Architecture Proposal

### Overview

This section presents the network architecture of Meridian FinServe Pvt. Ltd. in two states: the vulnerable **current state (before)** observed during the Trickbot + Cobalt Strike incident, and the **proposed hardened state (after)** designed to remediate the identified security gaps.

---

### Before — Current State (Vulnerable)

> See: [`architecture/before.svg`](architecture/before.svg)

The diagram depicts Meridian FinServe's network architecture at the time of the Trickbot + Cobalt Strike intrusion. The environment had no meaningful defence-in-depth. Key weaknesses observed:

- **No egress filtering** — ports 80, 443, and 447 were open outbound, allowing C2 beaconing to `192.236.155.230` (Trickbot) and `45.229.71.211` (Cobalt Strike) to proceed undetected.
- **No DNS firewall** — malicious domains such as `antivirusupdaty.com` resolved freely from inside the network.
- **Flat LAN (10.5.26.0/24)** — no east-west segmentation, allowing the Domain Controller (`VICTORYPUNK-DC`, 10.5.26.4) to reach all workstations over SMB without restriction.
- **No NDR / SIEM** — the infection went undetected for approximately 80 minutes.
- **No WAF** — the customer portal was exposed directly, with SQLi and IDOR vulnerabilities unpatched.
- **No MFA / PAM** — privileged access was uncontrolled.

**Observed Attack Chain:**

| Step | Event |
|------|-------|
| ① | Trickbot infection lands on `DESKTOP-OG16DGY` (10.5.26.132) — Patient Zero |
| ② | Public IP recon via `api.ip.sb`, `ipify.org`, `ipinfo.io` |
| ③ | C2 beacon to `192.236.155.230:80` (WinHTTP loader/1.0) |
| ④ | Trickbot gtag `rob87` POST → `36.95.27.243:443` |
| ⑤ | DC compromise via DCERPC/svcctl + Kerberos |
| ⑥ | Cobalt Strike beacon → `45.229.71.211` (~207s intervals) |
| ⑦ | SMB lateral movement: DC → 5 workstations |
| ⑧ | Data exfiltration via `antivirusupdaty.com` over port 447 |

### After — Proposed Hardened State

> See: [`architecture/after.svg`](architecture/after.svg)

The hardened architecture is structured across four defence layers, with each control directly mapped to an IOC or gap identified during the incident analysis.

| Layer | Before | After | IOC Addressed |
|-------|--------|-------|---------------|
| **Identity** | No MFA, no PAM | SSO + MFA + PAM + 8h session limit | IP-recon frames 809, 6801 |
| **Perimeter** | All ports open, no DNS FW, no WAF | Egress allowlist (80/443 only) + DNS firewall + WAF | C2 beaconing frame 11430 |
| **Segmentation** | Flat LAN, DC ↔ WS SMB unrestricted | VLAN 10/20/30 + default DENY east-west + Jump-host | SMB lateral movement frame 11100 |
| **Application** | SQLi, IDOR, no output encoding | Parameterised queries + authZ + CSP headers | Trigger B (portal) |
| **Data** | No at-rest encryption, no tokenisation | AES-256 at rest + tokenised PII + HSM for card data | 180K borrower records exposure |
| **Observability** | No NDR, no SIEM, 80 min blind | Suricata + Zeek + SIEM pipeline + alerts | Detection delay |
| **Response** | No runbook, no isolation playbook | Runbook + isolation playbook + post-incident retro | — |

---

### Risk Reduction Summary

| Threat | Before | After |
|--------|--------|-------|
| C2 beaconing | Unblocked | Blocked at perimeter (DNS FW + egress allowlist) |
| Lateral movement | Unrestricted SMB | Contained within VLANs |
| Detection time | ~80 minutes | < 5 minutes (SIEM alert) |
| Data exfiltration | Port 447 open | Port 447 blocked |
| Privileged access | No controls | PAM + MFA enforced |
