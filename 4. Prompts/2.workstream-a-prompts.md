## A.2 — Triage Pass (High-Level Inventory)

```
I have a PCAP file for Project KAVACH Workstream A (Meridian FinServe network forensics scenario). I need to produce a high-level triage inventory.

First, give me a sequence of tshark commands (Kali Linux) to extract:
1. Protocol hierarchy (-q -z io,phs)
2. Top conversations by bytes (-q -z conv,ip / conv,tcp)
3. Endpoint list with packet/byte counts (-z endpoints,ip)
4. Capture time bounds (capinfos or tshark -q -z io,stat)
5. Any bursts of activity over time (io,stat,10 or similar, 10-second intervals)
6. DNS query list (-Y dns -T fields -e dns.qry.name)

I'll run these and paste the outputs back to you.

Then, once I paste the raw output, turn it into a "A.2 Triage Pass" markdown section with:
- A summary table: protocols seen, top talkers (IP, role guess — client/server/internal/external), time range
- Flagged anomalies: any bursts, unusual ports, or endpoints worth investigating further
- A 3-4 sentence "first impressions" paragraph framing what this might represent in the Meridian FinServe network (internal segment, DMZ, etc.)

Don't speculate about the final verdict yet — that's for A.3.
```

Format as a markdown table or structured subsections, ready for report.md under "A.3 Hypothesis-Driven Deep Dive". The deliverable is the REASONING PROCESS, not just a final answer — so keep each hypothesis genuinely competing/plausible, not strawmen.

## A.4 — Indicator Extraction (IOC List)

```
Continuing Project KAVACH Workstream A (Meridian FinServe). I've confirmed the verdict from A.3: [paste 1-2 line verdict summary].

Now I need a structured, machine-readable IOC extraction. Help me in two steps:

STEP 1 — give me tshark commands to extract:
- Suspicious destination IPs/domains (with -e ip.dst, -e dns.qry.name filtered to non-baseline traffic)
- TLS JA3/JA3S fingerprints if TLS present (tshark with ja3 Lua script or tshark -z conv if not available — tell me what's feasible)
- HTTP User-Agent strings (-e http.user_agent)
- URI patterns (-e http.request.uri)
- Any file objects exportable via tshark --export-objects for hash reconstruction (sha256sum)

STEP 2 — once I paste back the raw values, produce a CSV with these exact columns:
indicator_type, indicator_value, first_seen_packet_no, last_seen_packet_no, confidence(High/Medium/Low), justification

Confidence rules: High = directly tied to confirmed malicious behavior from A.3 verdict; Medium = suspicious but ambiguous (e.g., shared infra); Low = anomalous but plausibly benign.

Output the CSV content in a code block so I can save it directly as iocs.csv — no prose embedded in the CSV itself.
```
